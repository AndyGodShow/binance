# 历史数据获取功能使用指南

## 概述

已成功集成币安历史K线数据获取功能，支持策略回测和数据分析。

## 功能特性

### ✅ 已完成

1. **历史K线API** (`/api/backtest/klines`)
   - 支持所有币安合约交易对
   - 支持多个时间周期（1m, 5m, 15m, 30m, 1h, 4h, 1d等）
   - 单次请求最多1500条数据
   - 自动处理时间范围过滤

2. **批量数据获取工具** (`historicalDataFetcher`)
   - 自动分页，突破1500条限制
   - 支持自定义时间范围
   - 预设时间范围（1天、7天、30天、90天、180天、1年）
   - 智能请求延迟，避免触发API限制

3. **数据测试界面**
   - 可视化配置界面
   - 实时数据预览
   - 数据统计信息

## 使用方法

### 1. 直接API调用

```typescript
// 获取最近500条1小时K线
const response = await fetch('/api/backtest/klines?symbol=BTCUSDT&interval=1h&limit=500');
const result = await response.json();
console.log(result.data); // KlineData[]
```

### 2. 使用工具类（推荐）

```typescript
import { historicalDataFetcher, HistoricalDataFetcher } from '@/lib/historicalDataFetcher';

// 方案A：获取预设时间范围的数据
const { startTime, endTime } = HistoricalDataFetcher.getPresetRange('30d');
const klines = await historicalDataFetcher.fetchRangeData(
    'BTCUSDT',
    '1h',
    startTime,
    endTime
);

// 方案B：获取最近N条数据
const recentData = await historicalDataFetcher.fetchRecentData('ETHUSDT', '15m', 1000);

// 方案C：自定义时间范围
const { startTime, endTime } = HistoricalDataFetcher.getTimeRange(
    '2024-01-01',
    '2024-12-31'
);
const customData = await historicalDataFetcher.fetchRangeData(
    'SOLUSDT',
    '4h',
    startTime,
    endTime
);
```

### 3. K线数据结构

```typescript
interface KlineData {
    openTime: number;              // 开盘时间（毫秒时间戳）
    open: string;                  // 开盘价
    high: string;                  // 最高价
    low: string;                   // 最低价
    close: string;                 // 收盘价
    volume: string;                // 成交量（币）
    closeTime: number;             // 收盘时间（毫秒时间戳）
    quoteVolume: string;           // 成交额（USDT）
    trades: number;                // 成交笔数
    takerBuyVolume: string;        // 主动买入成交量
    takerBuyQuoteVolume: string;   // 主动买入成交额
}
```

## 性能建议

1. **选择合适的时间周期**
   - 日内回测：使用 5m 或 15m
   - 波段回测：使用 1h 或 4h
   - 长期回测：使用 1d

2. **合理设置时间范围**
   - 避免一次性获取超过10万条数据
   - 大数据量建议分批获取或使用更大的时间周期

3. **缓存机制**
   - 历史数据不变，建议在客户端缓存
   - 可使用 localStorage 或 IndexedDB
